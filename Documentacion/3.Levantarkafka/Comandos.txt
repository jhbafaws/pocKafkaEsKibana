Ejecuta el siguiente comando para levantar los servicios de Kafka y Zookeeper

docker-compose up -d

Verifica que los contenedores están corriendo con:

1. Crear un topic

docker exec -it 5kafka-docker-kafka-1 kafka-topics --create --topic finytec-topic --bootstrap-server localhost:9092 --partitions 5 --replication-factor 1

docker exec -it 5ms-kafka-1 kafka-topics --create --topic finytec-topic --bootstrap-server localhost:9092 --partitions 5 --replication-factor 1

2. Listar los topics

docker exec -it 4kafka-docker-kafka-1 kafka-topics --list --bootstrap-server localhost:9092

docker exec -it 5ms-kafka-1 kafka-topics --list --bootstrap-server localhost:9092

3. Producir mensajes

docker exec -it 4kafka-docker-kafka-1 kafka-console-producer --topic finytec-topic --bootstrap-server localhost:9092

docker exec -it 4kafka-docker-kafka-1 kafka-console-producer --topic finytec-topic --bootstrap-server localhost:9092



4. Consumir mensajes

docker exec -it 4kafka-docker-kafka-1 kafka-console-consumer --topic finytec-topic --from-beginning --bootstrap-server localhost:9092

docker exec -it 5ms-kafka-1 kafka-console-consumer --topic finytec-topic --from-beginning --bootstrap-server localhost:9092

- Con PROPERTY:

docker exec -it 4kafka-docker-kafka-1 kafka-console-consumer --topic finytec-topic --from-beginning --bootstrap-server localhost:9092 --property print.key=true --property key.separator="-"

docker exec -it 5ms-kafka-1 kafka-console-consumer --topic finytec-topic --from-beginning --bootstrap-server localhost:9092 --property print.key=true --property key.separator="-"

Paso 5: Detener Kafka y Zookeeper

docker-compose down


ENVIAR MENSAJE

kafka-console-producer --broker-list localhost:9092 --topic customer_topic

kafka-topics --bootstrap-server localhost:9092 --delete --topic customer_topic
